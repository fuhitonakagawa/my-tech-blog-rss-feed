# ルール

## フィード追加のルール

ユーザーにフィードを追加してと言われたら、以下の手順でフィードを追加してください

1. フィード追加の最低限の情報をユーザーに聞く
  - 企業名・ブログ名
  - RSSフィードURL
  - 追加先のセクション（`src/resources/sections/` にある既存セクションを提示して選んでもらう。該当がなければ新規セクションの ID と表示名を確認する）
2. ブランチ作成
   `git checkout -b chore/new-feed-<企業名の英語>`
3. フィードを追加
   `src/resources/sections/<セクションID>.json` の `feeds` に `{ "label": "フィード名", "url": "フィードURL" }` を追加
   （新規セクションの場合は新しい JSON ファイルを作成。ファイル名がセクション ID になり、英小文字・数字・ハイフンのみ）
4. コミット
   `git commit -am 'chore(feed): <企業名など> 追加`
5. プッシュ
   `git push origin chore/new-feed-<企業名の英語>`
6. プルリクを作成
   pull_request_template.md を参考にプルリクを作成してください
   ghコマンドがあればそれを使用し、なければユーザーに作成方法を指示してください

すべての手順は都度ユーザーに同意を得て進めてください

## セクション追加のルール

ユーザーにセクションを追加してと言われたら、以下を確認して `src/resources/sections/<セクションID>.json` を作成してください

- セクション ID（URLスラッグ。英小文字・数字・ハイフンのみ。ファイル名になり、`/rss/<id>/` と `/rss/<id>/feeds/` に使われる）
- 表示名（`title`。ナビゲーション・見出し・フィードタイトルに使われる）
- 表示順（`order`。小さいほどナビゲーションの先頭に来る。既存セクションの値を参考に10刻みで設定）
- 所属させるフィード（`feeds`）

JSON の形式は既存のセクションファイルに合わせてください。
セクションページ・フィード（atom/rss/json）・ナビゲーション・sitemap は自動生成されるため、個別の追加作業は不要です
